# SessiaAval - Система управления салоном красоты

## Описание проекта

SessiaAval - это десктопное приложение для управления салоном красоты, разработанное на платформе Avalonia UI с использованием .NET 9 и PostgreSQL. Приложение предоставляет полный функционал для управления услугами, мастерами, клиентами и записями.

## Технологии

- **Frontend**: Avalonia UI 11.0.0
- **Backend**: .NET 9.0
- **База данных**: PostgreSQL 16
- **ORM**: Entity Framework Core 9.0.0
- **Тестирование**: nUnit

## Системные требования

- .NET 9.0 SDK или выше
- PostgreSQL 16 или выше
- Windows 10/11, Linux или macOS
- Минимум 4 GB RAM
- 500 MB свободного места на диске

## Установка и настройка

### 1. Установка PostgreSQL

Скачайте и установите PostgreSQL с официального сайта: https://www.postgresql.org/download/

При установке запомните:
- Порт: 5432 (по умолчанию)
- Пароль для пользователя postgres

### 2. Создание базы данных

Откройте pgAdmin или psql и выполните:

```sql
CREATE DATABASE matye_db;
```

### 3. Настройка строки подключения

Откройте файл `SessiaAval/Data/AppDbContext.cs` и проверьте строку подключения:

```csharp
"Host=localhost;Port=5432;Database=matye_db;Username=postgres;Password=ВАШ_ПАРОЛЬ"
```

Замените `ВАШ_ПАРОЛЬ` на пароль вашего PostgreSQL.

### 4. Создание схемы базы данных

Выполните SQL скрипты вручную через pgAdmin или psql:

```bash
# Подключитесь к базе данных
psql -U postgres -d matye_db

# Выполните скрипт создания схемы
\i Database/schema.sql

# Загрузите тестовые данные
\i Database/test_data.sql
```

Или через pgAdmin:
1. Откройте Query Tool
2. Откройте файл `Database/schema.sql` и выполните
3. Откройте файл `Database/test_data.sql` и выполните

### 5. Запуск приложения

```bash
cd SessiaAval
dotnet run
```

Или откройте решение `SessiaAval.sln` в Visual Studio/Rider и запустите проект.

## Руководство пользователя

### Регистрация и вход

1. При запуске приложения откроется окно входа
2. Для входа введите email и пароль, нажмите "ВОЙТИ"
3. Для регистрации нажмите "РЕГИСТРАЦИЯ", заполните форму:
   - ФИО
   - Email (уникальный)
   - Пароль
   - Телефон

### Работа с балансом (Пользователь)

1. В личном кабинете отображается текущий баланс
2. Для пополнения нажмите "ПОПОЛНИТЬ БАЛАНС"
3. Введите данные карты:
   - Номер карты (16 цифр)
   - Срок действия (MM/YY)
   - CVV (3 цифры)
   - Сумма пополнения
4. История всех операций отображается в таблице транзакций

### Запись на услугу (Пользователь)

1. Перейдите в раздел "ЗАПИСАТЬСЯ"
2. Выберите услугу из списка
3. Выберите мастера
4. Нажмите "ЗАПИСАТЬСЯ"
5. Вы получите номер в очереди
6. Оплата спишется автоматически после завершения услуги мастером

### Написание отзыва (Пользователь)

1. Перейдите в раздел "ОТЗЫВЫ"
2. Нажмите "НАПИСАТЬ ОТЗЫВ" напротив завершенной услуги
3. Выберите оценку (1-5 звезд)
4. Напишите комментарий
5. Нажмите "ОТПРАВИТЬ"

### Работа с клиентами (Мастер)

1. В разделе "МОИ КЛИЕНТЫ" отображаются все записи
2. Статистика: всего записей, активные, завершенные
3. Для завершения записи нажмите "Завершить"
4. Система автоматически:
   - Проверит баланс клиента
   - Спишет стоимость услуги
   - Изменит статус на "completed"

### Просмотр отзывов (Мастер)

1. Перейдите в раздел "ОТЗЫВЫ"
2. Отображается статистика:
   - Общее количество отзывов
   - Средняя оценка
3. Таблица со всеми отзывами: дата, клиент, услуга, оценка, комментарий

### Управление квалификацией (Мастер)

1. Перейдите в раздел "КВАЛИФИКАЦИЯ"
2. Отображается текущий уровень квалификации
3. Для повышения нажмите "Подать заявку на повышение"
4. Заявка будет рассмотрена модератором

### Управление услугами (Модератор)

1. В разделе "УСЛУГИ" отображаются все услуги салона
2. Для добавления услуги:
   - Нажмите "ДОБАВИТЬ УСЛУГУ"
   - Заполните: название, категория, цена, длительность, описание
3. Для редактирования нажмите "Изменить" напротив услуги
4. Для назначения мастера:
   - Нажмите "Назначить мастера"
   - Выберите мастера из списка

### Управление мастерами (Модератор)

1. В разделе "МАСТЕРА" отображаются все мастера
2. Для одобрения заявки на повышение:
   - Найдите мастера с индикатором "Заявка на повышение"
   - Нажмите "Одобрить заявку"
3. Для ручного повышения:
   - Нажмите "Повысить" напротив мастера
   - Уровень квалификации увеличится на 1

### Управление пользователями (Администратор)

1. В разделе "ПОЛЬЗОВАТЕЛИ" отображаются все клиенты
2. Для добавления пользователя нажмите "ДОБАВИТЬ ПОЛЬЗОВАТЕЛЯ"
3. Для редактирования нажмите "Изменить"
4. Для удаления нажмите "Удалить"
   - Система покажет количество связанных записей
   - При подтверждении удалятся все связанные данные (каскадное удаление)

### Управление сотрудниками (Администратор)

1. В разделе "СОТРУДНИКИ" отображаются мастера и модераторы
2. Для добавления сотрудника:
   - Нажмите "ДОБАВИТЬ СОТРУДНИКА"
   - Выберите роль (Мастер или Модератор)
   - Заполните данные
3. Для редактирования нажмите "Изменить"
4. Для удаления нажмите "Удалить"

## Особенности системы

### Автоматическая оплата

При завершении записи мастером:
1. Система проверяет баланс клиента
2. Если средств достаточно - списывает стоимость услуги
3. Создает запись в истории транзакций
4. Меняет статус записи на "completed"
5. Если средств недостаточно - показывает ошибку, запись остается активной

### Каскадное удаление

При удалении пользователя автоматически удаляются:
- Все его записи на услуги
- История транзакций баланса
- Написанные отзывы
- Профиль мастера (если пользователь был мастером)

### Валидация данных

**Пополнение баланса:**
- Номер карты: ровно 16 цифр
- Срок действия: формат MM/YY
- CVV: ровно 3 цифры
- Сумма: больше 0

**Регистрация:**
- Email: уникальный, корректный формат
- Пароль: минимум 3 символа
- Телефон: обязательное поле

## Устранение неполадок

### Ошибка подключения к БД

**Проблема:** "Connection refused" или "password authentication failed"

**Решение:**
1. Проверьте, что PostgreSQL запущен
2. Проверьте строку подключения в `AppDbContext.cs`
3. Убедитесь, что база данных `matye_db` создана
4. Проверьте пароль пользователя postgres

### Ошибка при применении миграций

**Проблема:** "A network-related or instance-specific error"

**Решение:**
1. Убедитесь, что PostgreSQL работает на порту 5432
2. Проверьте firewall
3. Попробуйте подключиться через pgAdmin

### Приложение не запускается

**Проблема:** Ошибки компиляции или runtime ошибки

**Решение:**
1. Убедитесь, что установлен .NET 9.0 SDK
2. Выполните `dotnet restore`
3. Очистите и пересоберите проект: `dotnet clean && dotnet build`

### Не работает вход

**Проблема:** "Неверный email или пароль"

**Решение:**
1. Проверьте, что тестовые данные загружены
2. Используйте тестовые аккаунты с паролем "123"
3. Проверьте таблицу users в БД
