## 3. Информация о базе данных

### Структура базы данных

**Количество таблиц:** 9

**Список таблиц:**

1. **roles** - Роли пользователей
2. **users** - Пользователи системы
3. **masters** - Профили мастеров
4. **service_categories** - Категории услуг
5. **services** - Услуги салона
6. **master_services** - Связь мастеров и услуг (многие-ко-многим)
7. **appointments** - Записи клиентов на услуги
8. **reviews** - Отзывы о мастерах и услугах
9. **balance_transactions** - История операций с балансом

### Описание основных таблиц

#### Таблица users
**Назначение:** Хранение информации о всех пользователях системы

**Поля:**
- user_id (PK) - Уникальный идентификатор
- role_id (FK) - Ссылка на роль
- email (UNIQUE) - Email для входа
- password_hash - Хеш пароля
- first_name - Имя
- last_name - Фамилия
- phone - Номер телефона
- balance - Баланс счета (decimal)
- registration_date - Дата регистрации
- last_modified - Дата последнего изменения

**Связи:**
- → roles (многие к одному)
- ← masters (один к одному)
- ← appointments (один ко многим)
- ← reviews (один ко многим)
- ← balance_transactions (один ко многим)

#### Таблица services
**Назначение:** Каталог услуг салона

**Поля:**
- service_id (PK) - Уникальный идентификатор
- category_id (FK) - Ссылка на категорию
- service_name - Название услуги
- description - Описание
- price - Стоимость (decimal)
- duration_minutes - Длительность в минутах
- is_active - Активна ли услуга
- created_date - Дата создания
- last_modified - Дата изменения

**Связи:**
- → service_categories (многие к одному)
- ← appointments (один ко многим)
- ← master_services (один ко многим)

#### Таблица appointments
**Назначение:** Записи клиентов на услуги

**Поля:**
- appointment_id (PK) - Уникальный идентификатор
- user_id (FK) - Клиент
- master_id (FK) - Мастер
- service_id (FK) - Услуга
- appointment_date - Дата и время записи
- queue_number - Номер в очереди
- status - Статус (pending/completed)
- created_date - Дата создания
- last_modified - Дата изменения

**Связи:**
- → users (многие к одному)
- → masters (многие к одному)
- → services (многие к одному)

### Индексы

**Созданные индексы:**
- users.email (UNIQUE) - Для быстрого поиска по email
- appointments.user_id - Для выборки записей пользователя
- appointments.master_id - Для выборки записей мастера
- appointments.appointment_date - Для сортировки по дате
- reviews.master_id - Для выборки отзывов мастера

### Ограничения целостности

**Foreign Keys с каскадным удалением:**
- users → appointments (CASCADE DELETE)
- users → reviews (CASCADE DELETE)
- users → balance_transactions (CASCADE DELETE)
- masters → appointments (CASCADE DELETE)

**Проверочные ограничения:**
- balance >= 0 (баланс не может быть отрицательным)
- rating BETWEEN 1 AND 5 (оценка от 1 до 5)
- price > 0 (цена должна быть положительной)

### Размер базы данных

**Тестовые данные:**
- Пользователей: ~10
- Мастеров: 3
- Услуг: ~15
- Записей: ~20
- Отзывов: ~10

**Ожидаемый размер БД:** ~5-10 MB с тестовыми данными

## 5. Информация о программе

### Назначение программы

SessiaAval - это комплексная система управления салоном красоты, предназначенная для автоматизации процессов записи клиентов, управления услугами, мастерами и финансовыми операциями.

### Целевая аудитория

1. **Клиенты салона** - Запись на услуги, управление балансом
2. **Мастера** - Просмотр расписания, завершение услуг
3. **Модераторы** - Управление услугами и мастерами
4. **Администраторы** - Полное управление системой

### Основные функции

#### Для клиентов:
- Регистрация и авторизация
- Просмотр каталога услуг
- Запись на услуги с выбором мастера
- Управление балансом (пополнение, история)
- Просмотр своих записей и номера в очереди
- Написание отзывов о завершенных услугах

#### Для мастеров:
- Просмотр списка клиентов и записей
- Завершение услуг с автоматическим списанием оплаты
- Просмотр назначенных услуг
- Управление квалификацией (подача заявок)
- Просмотр отзывов о своей работе

#### Для модераторов:
- Управление каталогом услуг (CRUD)
- Назначение услуг мастерам
- Управление квалификацией мастеров
- Одобрение заявок на повышение квалификации

#### Для администраторов:
- Управление пользователями (CRUD)
- Управление сотрудниками (мастера, модераторы)
- Просмотр всех записей и статистики
- Каскадное удаление пользователей

### Ключевые особенности

**Автоматизация оплаты:**
- Оплата списывается автоматически при завершении услуги
- Проверка достаточности средств
- Создание транзакций в истории

**Система очередей:**
- Автоматическое присвоение номера в очереди
- Отслеживание статуса записи

**Управление квалификацией:**
- Система заявок на повышение
- Уровни квалификации мастеров

**Безопасность:**
- Ролевая модель доступа
- Валидация данных
- Защита от SQL-инъекций (через EF Core)

### Ограничения текущей версии

1. Нет функции отмены записи клиентом
2. Нет системы уведомлений
3. Нет возможности выбора конкретного времени (только слоты)
4. Пароли хранятся в открытом виде (для учебных целей)
5. Нет функции восстановления пароля

### Планы развития

**Версия 1.1:**
- Добавление системы уведомлений
- Возможность отмены записи
- Хеширование паролей (bcrypt)

**Версия 1.2:**
- Отчеты и аналитика
- Экспорт данных в Excel
- Резервное копирование

**Версия 2.0:**
